# CLAUDE.md — أداة مزامنة المستودع مع متجر زد (ASJ / zid-sync)

أداة صفحة واحدة (`index.html`) تعمل بالكامل داخل المتصفح (static site على GitHub Pages، مستودع `mrbadraljohani-gif/zid-sync`). لا يوجد خادم؛ كل المعالجة JavaScript في الصفحة، والمكتبة الوحيدة هي SheetJS (xlsx) لقراءة/كتابة إكسل.

## المدخلات
- **ملف المستودع**: يحوي أعمدة «رقم الصنف»، «الكمية»، أسعار («شامل»/«قبل الضريبة»)، «اسم».
- **ملف تصدير زد**: تصدير المنتجات الكامل (~76 عموداً)، أهمها: `sku`, `name_ar`, `name_en`, `quantity`, `price`, `barcode`, وعمود صور (يُكشف تلقائياً).

## المطابقة
لكل صنف في زد يُبحث عن مطابقته بالمستودع بالترتيب: (1) ربط يدوي محفوظ `manualMap[sku]`، (2) تطابق الكود، (3) كود الأساس قبل النقطة. الأصناف بلا مطابقة تظهر في «يحتاج قرار» للربط اليدوي (يُحفظ ويُعاد استخدامه تلقائياً). المطابقة بالاسم مستخدمة فقط لاقتراحات الربط اليدوي (لا في الإخراج).

## آلية الإخراج: ملفّان منفصلان (مؤكدة بالتجربة مع استيراد زد)

### 1) ملف الكميات — `zid_quantities_update.xlsx`
- **٤ أعمدة حرفياً وبالترتيب**: `sku | name_ar | name_en | <ZID_QTY_COL>`
- `ZID_QTY_COL` ثابت (const) أعلى السكربت = `"عبدالله صلاح الجهني للتجارة - كميات"` — **غيّره إذا تغيّر اسم المستودع في زد**.
- صف لكل صنف مطابق تغيّرت كميته؛ العمود الرابع = الكمية الحقيقية من المستودع (عدد صحيح ≥ 0).
- **«غير المحدود» (Infinite) المطابِق يُستبدل بالكمية الحقيقية** (يحل مشكلة الأصناف غير المحدودة). غير المحدود بلا مطابقة يظهر في بطاقة «غير محدود بلا مطابقة».
- `name_ar`/`name_en` تُنسخ من تصدير زد حرفياً.
- **مسار الاستيراد في زد**: استيراد ← تحديث كميات المنتجات الحالية فقط.

### 2) ملف الأسعار — `zid_prices_update.xlsx`
- **نفس بنية تصدير زد كاملة** (كل الأعمدة بترتيبها وتسمياتها)، يُعدَّل عمود `price` فقط.
- قاعدة السعر: تجاهل فرق < 1 ريال، وتقريب قيمة المستودع لعدد صحيح.
- **عمود `quantity` يبقى بقيمه الأصلية دون أي لمس** (الكميات مسؤولية الملف الأول).
- لتقليل الحجم: يشمل فقط صفوف المنتجات التي تغيّر سعرها فعلاً + صف الرأس الكامل.
- **مسار الاستيراد في زد**: استيراد ← تحديث المنتجات الحالية ← حدّد حقل السعر فقط ← تجاهل الحقول الفارغة.

## صمامات الأمان (تمنع التحميل عند الفشل)
- **ملف الكميات** (`verifyQuantities`): الرأس = ٤ أعمدة بالتسميات الحرفية بالترتيب، `sku` غير فارغ، الكميات أعداد صحيحة ≥ 0.
- **ملف الأسعار** (`verifyPrices`): أعمدة مطابقة للأصل بالترتيب والتسميات، ولا يتغيّر سوى عمود `price` (فيبقى `sku`/`barcode`/`quantity` وبقية الأعمدة كما هي).

## الواجهة
- بطاقات النتائج: «تم تحديثه» (تغيّرت كميته أو سعره، ويشمل تحويلات غير المحدود)، «يحتاج قرار»، «بدون ربط»، «غير محدود بلا مطابقة»، «إجمالي الأسطر».
- زرّا تحميل منفصلان مع عدّاد لكل ملف وسطر تعليمات الاستيراد أسفله؛ سطر التأكيد الأخضر يفصّل عددَي الكميات والأسعار.

## الحفظ والإعدادات (localStorage + الريبو)
- الروابط `mapping.json` والمتجاهلات `ignored.json` تُحفظ في الريبو مباشرةً عبر GitHub Contents API (زر «☁ الحفظ في الريبو» بالترويسة)، تلقائياً بعد تحميل أي ملف. رمز GitHub (fine-grained, Contents R/W على `zid-sync`) يُخزَّن في localStorage لكل جهاز — **لا يُوضع في الكود إطلاقاً** (الموقع عام).
- وضع فاتح/غامق (`zid-theme`)، خيارات (`zid-options-v1`)، استبعادات (`zid-excluded-v1`).
