# CLAUDE.md

إرشادات العمل على هذا المشروع لـ Claude Code. اقرأ هذا الملف قبل أي تعديل.

## وظيفة الأداة

أداة (صفحة ويب واحدة) لمزامنة مخزون المستودع مع متجر **زد**. المستخدم يرفع ملفين Excel:

1. **ملف المستودع** — فيه رقم الصنف (الكود)، الكمية، السعر (شامل الضريبة / قبل الضريبة)، والاسم.
2. **ملف تصدير منتجات زد** — يحتوي على أعمدة زد القياسية (`sku`, `barcode`, `quantity`, `price`, `name_ar`, …).

تطابق الأداة أصناف زد مع أصناف المستودع، ثم تنتج **ملف Excel محدّثاً** يُستورَد في زد. كل المعالجة تتم في متصفح المستخدم — لا تُرفع الملفات لأي خادم. تُحفظ إعدادات الربط والتجاهل في `localStorage` وتُرفع إلى الريبو عبر GitHub API لمشاركتها بين الأجهزة.

## بنية index.html

ملف واحد يحوي HTML + CSS + JavaScript، بالعربية واتجاه RTL. أقسامه بالترتيب:

- **`<head>`** — الخطوط، مكتبة `xlsx` من CDN، سكربت مبكّر يطبّق الوضع الفاتح/الغامق من `localStorage`، ثم `<style>` كبير قائم على متغيّرات CSS (`:root` = الوضع الغامق الافتراضي، و`:root[data-theme="light"]` = الوضع الفاتح).
- **الترويسة (topbar)** — الشعار والاسم، زر تبديل الوضع، وقائمة GitHub المنسدلة (`ghPanel`) لإدخال الرمز مع مؤشر حالة (`gh-dot`: أخضر=محفوظ، كهرماني=تحذير، أحمر=فشل).
- **hero** — عنوان تعريفي.
- **القسم ١ — ارفع الملفات** (`~سطر 421`): رفع ملف المستودع وملف زد.
- **القسم ٢ — اضبط الخيارات** (`~سطر 439`): `price` (شامل الضريبة `incl` / قبل الضريبة `excl` / لا تحدّث `none`)، و`inf` لمنتجات الكمية غير المحدودة (اتركها `skip` / حدّثها `update`).
- **زر المطابقة + بطاقة النتيجة** (`#result`): إحصاءات (محدّث / جديد / بدون ربط / غير محدود / الإجمالي)، زر تحميل الملف، رسالة التحقق (`verifyMsg`)، وجدول التفاصيل القابل للفلترة.
- **القسم ٣ — ربط يدوي** (`~سطر 481`): بحث في المستودع، إضافة ربط `SKU زد ← كود المستودع`، وبطاقة المتجاهَلات.
- **footer** ثم **`<script>`** (`~سطر 512`): متغيّرات الحالة ومفاتيح `localStorage`، \u200F`loadConfig()`، دوال المطابقة (\u200F`normCode`, `suggestFor`, `priceInt`, …)، \u200F`run()` (قلب المعالجة)، \u200F`verifyOutput()` (صمام الأمان)، الحفظ في GitHub (\u200F`GH`, `putFileToRepo`, `saveToRepo`)، ودوال عرض الواجهة.

## mapping.json و ignored.json

- **`mapping.json`** — كائن (object) يربط `SKU زد ← كود المستودع` للأصناف التي يختلف كودها بين النظامين. مثال: `{ "80207": "080207" }`.
- **`ignored.json`** — مصفوفة (array) من أكواد أصناف زد المتجاهَلة (لا تظهر في قائمة «بدون ربط»).

كلاهما يُحمَّل عبر `fetch` مع دمجه بنسخة `localStorage` (يتعامل مع 404 بهدوء)، ويُرفع إلى الريبو عبر GitHub API. إعداد الريبو في `const GH = { owner: "mrbadraljohani-gif", repo: "zid-sync", branch: "main" }`.

## قواعد العمل (مهمة — لا تخالفها)

1. **الملف الناتج يحافظ على جميع الأعمدة، ولا يغيّر سوى الكمية (`quantity`) والسعر (`price`).** أي تغيير في أي عمود آخر (خاصة `sku` و`barcode`) يُوقفه `verifyOutput()` ويمنع التحميل. لا تُلغِ هذا الفحص.
2. **كل الأسعار أعداد صحيحة** — تُقرَّب عبر `priceInt()` / `Math.round()`.
3. **تجاهل فرق السعر الأقل من ريال واحد** — لا يُحدَّث السعر إلا إذا كان الفرق `>= 1` ريال.
4. **ترتيب المطابقة:** الربط اليدوي (`mapping.json`) أولاً ← ثم SKU الكامل ← ثم الكود الأساسي (مع تجاهل الأصفار البادئة، مثل `080158` = `80158`).
5. منتجات «الكمية غير المحدودة» (`infinite`) تُترك كما هي افتراضياً (`opts.inf === "skip"`).
